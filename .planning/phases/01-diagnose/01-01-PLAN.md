<task type="auto">
  <name>Categorize errors and identify root causes</name>
  <files>None (analysis only)</files>
  <action>
Analyze the captured build output and categorize each error:

**Error Categories (from RESEARCH.md):**
1. **Module/Export Errors** - "Export X doesn't exist" or "Module has no exports"
   - Root cause: Empty component files (0 bytes)
   - Affected files: Components that are imported but not implemented

2. **ESLint/Syntax Errors** - "react/no-unescaped-entities" or type mismatches
   - Root cause: Unescaped HTML entities in JSX, unused variables
   - Affected files: Page files with content issues

3. **Runtime Constraint Errors** - Server-only APIs in static context
   - Root cause: Using `cookies()`, `headers()`, `getServerSideProps`, etc.
   - Affected files: Pages using SSR-only features

4. **Dependency/Import Errors** - Missing dependencies or incorrect import paths
   - Root cause: Module resolution issues or missing packages
   - Affected files: Files with broken imports

For each error found:
- Document the file path
- Document the line number
- Document the exact error message
- Identify the category
- Determine the root cause
- Note which pages are blocked by this error

Create a structured analysis that maps:
- Error → File → Line → Category → Root Cause → Affected Pages
  </action>
  <verify>
Each build error is categorized with identified root cause. Clear mapping exists from error to affected files and pages.
  </verify>
  <done>
All errors are categorized by type with root causes identified. Clear understanding of which pages are blocked by which errors.
  </done>
</task>
