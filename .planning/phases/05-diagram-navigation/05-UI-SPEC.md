## Layout & Interaction Specifications

### Diagram Panel (Left, Desktop Only)

**Breakpoint visibility:**
- Hidden below `lg` (1024px) — `hidden lg:block`
- Full height on desktop — `h-screen`
- Fixed width — `lg:w-[35%]`
- Sticky positioning — `sticky top-0` (remains in viewport as user scrolls content)
- Border right — `border-r border-gray-200`

**ReactFlow configuration (read-only):**
All of these props must be set to `false` to disable user interaction:
- `nodesDraggable={false}` — prevent node movement
- `nodesConnectable={false}` — prevent edge connection dragging
- `elementsSelectable={false}` — prevent click selection
- `zoomOnScroll={false}` — disable scroll zoom
- `zoomOnPinch={false}` — disable pinch zoom
- `zoomOnDoubleClick={false}` — disable double-click zoom
- `panOnDrag={false}` — disable click-drag pan
- `panOnScroll={false}` — disable scroll-based pan
- `selectNodesOnDrag={false}` — disable drag selection
- `nodesFocusable={false}` — disable keyboard navigation on nodes
- `edgesFocusable={false}` — disable keyboard navigation on edges

Keep handlers attached:
- `onNodesChange={onNodesChange}` — required for internal rendering
- `onEdgesChange={onEdgesChange}` — required for internal rendering

**Background & styling:**
- `fitView` prop enabled — auto-fit diagram to panel
- `attributionPosition="bottom-left"` — ReactFlow license badge
- Background color via `<Background color="#e5e7eb" gap={20} />`

### Content Panel (Right, Desktop 65%, Mobile 100%)

**Breakpoint sizing:**
- Full width on mobile — `w-full`
- 65% width on desktop — `lg:w-[65%]`
- Scrollable height — `overflow-y-auto`
- Default height on mobile, full screen on desktop — `h-auto lg:h-screen`

**Padding & spacing:**
- Horizontal padding — `px-8`
- Vertical padding — `py-24`
- Max content width — `max-w-xl mx-auto`

### GitHub Buttons

**Location 1: Header (near page title)**
- Positioned inline with title flexbox — `flex items-center justify-between`
- Uses semantic `<a>` tag (not `<button>`)
- Link props:
  - `href="https://github.com/d-aniel-yi/mobi"`
  - `target="_blank"`
  - `rel="noopener noreferrer"`
  - `aria-label="View Mobi repository on GitHub"`
- Styling:
  - Layout: `inline-flex items-center gap-2`
  - Padding: `px-4 py-2`
  - Colors: `bg-gray-900 text-white`
  - Typography: `text-sm font-mono`
  - Rounded corners: `rounded`
  - Hover state: `hover:bg-gray-700 transition-colors`
  - Whitespace: `whitespace-nowrap` (prevent text wrap on small screens)
- Icon: lucide-react `<Github className="w-4 h-4" />`
- Text: "View on GitHub"

**Location 2: Footer (end of content)**
- Positioned after all STEPS sections
- Separated by top border and padding: `mt-24 pt-8 border-t border-gray-200`
- Same link props and styling as header button
- Same icon and text

### Animation Loop

**Behavior:**
- Starts automatically on page load (no user trigger)
- Removes "Submit Job" button entirely
- Continuous looping: after sequence completes, immediately starts again

**Sequence timing** (matches existing simulation logic):
| Edge | Delay | Color | Event |
|------|-------|-------|-------|
| e1 (frontend→backend) | 500ms | #16a34a (green) | HTTP request |
| e2 (backend→redis) | 1500ms | #dc2626 (red) | Push Job |
| e3 (redis→worker) | 2500ms | #d97706 (amber) | Pop Job |
| e5 (worker→db) | 4000ms | #2563eb (blue) | Save Result |
| Loop reset | 4500ms | (default #b1b1b7) | Start next cycle |

**Implementation details:**
- Use `useEffect` with `useCallback` for animation loop
- `setEdges` to update animated state and stroke color
- `setTimeout` for staggered activation
- **CRITICAL:** Return cleanup function from useEffect to clear all pending timeouts when component unmounts
- After 4000ms, all edges reset to default (non-animated) at 4500ms before next loop starts

## Constraints & Caveats

1. **Sticky positioning requires flex layout without overflow on parent:** Do not apply `overflow-y-auto` to the outer flex container. Overflow must be on the content panel only (right side).

2. **Animation cleanup is mandatory:** useEffect must return a cleanup function that clears all pending timeouts. Failure to do so causes memory leaks and stale state updates on component unmount.

3. **ReactFlow handlers must remain attached:** Keep `onNodesChange` and `onEdgesChange` handlers connected to ReactFlow even though all interactions are disabled via props. Removing handlers breaks internal rendering.

4. **Mobile diagram is hidden, not resized:** Below lg breakpoint, diagram is `display: none`, not scaled down. Content becomes full-width.

5. **Attribution badge positioning:** ReactFlow's license badge with `attributionPosition="bottom-left"` may overlap content on smaller panels. Verify positioning after implementation.
