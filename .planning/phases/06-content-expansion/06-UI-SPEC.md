## Code Block Styling (Critical Pattern for Phase 6)

All new sections that include code snippets must use this exact pattern:

**Container:** 
```html
<div className="mt-6 p-4 bg-[#1e1e1e] text-gray-300 rounded-sm font-mono text-xs overflow-x-auto border border-gray-800">
```

**Content structure:**
- Wrap code lines in `<p>` elements (one per line)
- Indent with `&nbsp;` (not spaces; ensures proper rendering in JSX)
- Use `<span>` with color classes for syntax highlighting
- Use `<br />` for blank lines between logical sections
- Add comment line above code snippets: `<p className="text-gray-500"># Comment or filename</p>`

**Example from Phase 5 (template for Phase 6):**
```jsx
<div className="mt-6 p-4 bg-[#1e1e1e] text-gray-300 rounded-sm font-mono text-xs overflow-x-auto border border-gray-800">
    <p className="text-gray-500"># comment_or_filename</p>
    <p><span className="text-purple-400">keyword</span> value</p>
    <p>&nbsp;&nbsp;<span className="text-green-400">"string"</span></p>
    <br />
    <p className="text-gray-500"># another section</p>
    <p><span className="text-blue-400">function_name</span>()</p>
</div>
```

## Section Structure Pattern (Reused from Phase 5)

Each new section in the `STEPS` array must follow this exact structure:

```jsx
{
    id: "unique-identifier",
    title: "Section Title in Title Case",
    content: (
        <>
            <p>First paragraph explaining the topic.</p>
            <p className="mt-4">Second paragraph with more details.</p>
            
            <div className="mt-6 p-4 bg-[#1e1e1e] ...">
                {/* Code block here */}
            </div>
            
            <p className="mt-4">Paragraph after code block explaining the code.</p>
        </>
    ),
},
```

**Key points:**
- Content is JSX, not HTML strings
- Use `className="mt-4"` for paragraph spacing (16px)
- Use `className="mt-6"` for code block spacing (24px)
- Section content is wrapped in `<>`/`</>` (fragment)
- Paragraph text inherits parent styling (gray-600, text-sm, leading-relaxed)

## Constraints & Caveats

1. **Code block styling must match Phase 5 exactly:** Copy the `bg-[#1e1e1e]` div pattern verbatim. Variations in color, padding, or font will break visual consistency.

2. **Content is static JSX only:** Do not use dynamic imports, data fetching, or client-side rendering within section content. All code examples and text must be hardcoded in the STEPS array.

3. **No new typography tokens:** Do not introduce new font sizes or weights. Use only the 5 established roles from Phase 5.

4. **Spacing must use existing scale:** All gaps, padding, and margins must be multiples of 4px (Tailwind default). Do not create new spacing values.

5. **Color palette is locked:** Do not add new colors. Code syntax highlighting colors are predefined; use only the specified classes for new code examples.

6. **Section count flexibility:** Phase 6 adds at least one new section (CONT-01 requirement). Exact count is determined during planning; executor adds sections incrementally to STEPS array.
